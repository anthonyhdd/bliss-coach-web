# bliss-coach-web

SEO content platform for the Bliss Coach app portfolio, replacing the old Framer landing page at https://bliss-coach.com.

- **Stack**: Astro (static) + vanilla CSS, deployed to GitHub Pages via Actions. Free.
- **Structure**: `/` studio home · `/{sofia,alex,charm,amelie}/` app landing pages · `/<app>/blog/<slug>/` SEO articles.

## Adding articles (the whole point)

Drop a markdown file in `src/content/blog/<app>/<slug>.md`:

```yaml
---
title: "Max 70 chars, keyword-first"          # becomes H1 + <title>
description: "Max 165 chars meta description"
app: alex            # sofia | alex | charm | amelie
publishDate: 2026-07-02
keywords: ["primary keyword", "secondary"]
---

Body in markdown. H2/H3 only (no H1). End with an "## FAQ" section.
```

Merge to `main` → GitHub Actions builds and deploys automatically. Sitemap, canonical URLs, Article + Breadcrumb JSON-LD, OG tags, and the per-app Apple Smart Banner are generated by the templates.

Per-app branding (colors, App Store IDs, taglines) lives in `src/config/apps.ts`.

## Local dev

```
npm install
npm run dev        # localhost:4321
npm run build      # output in dist/
```

## Deploy states

1. **Preview (current)**: `BASE_PATH=/bliss-coach-web` in `.github/workflows/deploy.yml` → site at https://anthonyhdd.github.io/bliss-coach-web/
2. **Production cutover**:
   - Delete the `BASE_PATH` env block in the workflow
   - Add `public/CNAME` containing exactly `bliss-coach.com`
   - Repo Settings → Pages → Custom domain: `bliss-coach.com` (+ Enforce HTTPS once cert issues)
   - At Hover (registrar of bliss-coach.com): replace the Framer A records with GitHub Pages IPs `185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153`, and point the `www` CNAME to `anthonyhdd.github.io`
   - After propagation, cancel the Framer subscription
3. **Later (optional)**: add `pt=` / `ct=` campaign tokens to App Store links in `apps.ts` for per-article install attribution in App Store Connect.
